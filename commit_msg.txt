🔒 Security: Implement default-deny policy for Docker container control

🎯 **What:** The docker_control command previously allowed execution on any container name. This commit introduces allowed_containers settings at both the server and global levels to restrict which containers can be controlled.
⚠️ **Risk:** The vulnerability permitted unrestricted Docker container control to users who might only be intended to restart specific containers.
🛡️ **Solution:** Enforces a default-deny policy. Execution is only allowed if the requested container matches the allowlist in the respective server's settings (falling back to global feature settings).
